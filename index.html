How the dot cloud works

Every slider set defines one mean release. We fire 900 simulated shots with small Gaussian errors (σv, σθ, drift), fly each one with the exact projectile model, and mark where its center crosses the rim plane. Green = inside the make circle, red = miss. P(make) is simply the green fraction.

H − h = d·tanθ − g·d² / (2·v²·cos²θ)
θ* = 45° + ½·arctan((H − h) / d)
make ⇔ (depth² + lateral²) ≤ r²,  r = (D_rim − D_ball)/2 ≈ 10.9 cm
P(make) = (# landings inside circle) / N
Scope: verdict and cloud use the exact no-drag projectile model (unit-tested). The cyan arc adds quadratic drag (C_d ≈ 0.47) and Magnus lift from backspin (estimated C_L) — shown for comparison, not scored. Ball–rim contact is not simulated; backspin's biggest real-game payoff (softer bounces) happens at contact.
